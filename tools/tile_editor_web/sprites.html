      Welcome — sprite editor in 5 steps

      1. Make a sprite — click + New. A blank 2×2-tile sprite appears with empty cells (dashed orange boxes).
      2. Pick cell size — set Width × Height (1–8 tiles). Common Zelda-II sizes: 2×2 for enemies, 2×3 for the hero.
      3. Assign tiles — click an empty cell, then left-click a tile in the Shared tileset to drop it into the cell. Right-click a tileset tile to jump to the first sprite that already uses it. You can also hit 📥 Assign or press Enter.
      4. Switch to Paint mode — press M (or click ✏️ Paint) to edit pixels. In 🖱 Browse mode nothing gets painted by accident — clicks only select.
      5. Preview — click 🖥 Full preview (or press F) to see all sprites at real NES size.

      How clicks work

      • Tileset, left-click → assign that tile to the selected cell in the big sprite.
      • Tileset, right-click → jump to the first sprite using that tile (great for "where did I put that tile?").
      • Sprite, left-click → select the cell (in Paint mode also paints the pixel).
      • Sprite, right-click → stamp the highlighted tile into the clicked cell — fast tile assignment in any mode.

      Browse vs Paint

      • 🖱 Browse — the safe default. Clicking a cell in the big sprite just selects it; nothing gets painted by accident. Palette, flip, priority controls still work.
      • ✏️ Paint — clicks on the big sprite paint pixels. Tileset clicks behave the same in both modes. Switch with M.

      Everything auto-saves. Ctrl+Z (or just Z) undoes. The tileset and palettes are shared with the Backgrounds page.

      NES sprite rules (and why some choices exist)

      • Sprites CAN flip. Unlike background tiles, every sprite in OAM has flip-H and flip-V bits — no need to store a mirrored copy in a separate tile slot.
      • No rotation. The hardware has no rotate bit. A 90°-rotated tile must be a different tile in the CHR.
      • 64 sprites on-screen, 8 per scanline. Each cell in a sprite becomes one 8×8 hardware sprite — a 2×3 hero is 6 of your 64 sprite slots.
      • 4 sprite palettes (SP0–SP3). Each holds 3 colours — slot 0 is always transparent. Palette is assigned per cell.
      • Priority bit. "Behind BG" draws the cell underneath non-transparent background pixels — useful for hiding a character in bushes or doorways.
      • Shared CHR. Backgrounds and sprites share the same 256 tiles. Editing a sprite's tile can change any background that uses the same index.

      Saving & recovery

      Your work is saved to this browser's storage every time you change anything. Refreshing the page reloads the last-saved version — so you can close the tab and come back later without losing anything.

      • Every 30 seconds a numbered snapshot is kept (last 5 stay around).
      • Every 5 minutes an emergency backup is written to browser storage.
      • Recover… in the toolbar lets you roll back to any snapshot or emergency backup. The current project is snapshotted first, so nothing is ever lost in the process.
      • Export → JSON save gives you a portable file you can email, commit to git, or keep as a permanent backup. Import it back with Import sprites….
      • The Backgrounds and Sprites pages share one project in storage — changes on either page save into the same file.

      Tips & gotchas

      • "This tile is also used…" warning. The editor is telling you that painting here will change every sprite or background tile using that index. Click Duplicate first to get a fresh tile slot for just this cell.
      • Outlines around sprites look wrong. Check that colour 0 (the transparent colour) is set. Sprite colour 0 is ALWAYS transparent — you cannot use it to draw black.
      • Empty cell? Assign any tile (even the blank tile 0) with an outline palette. To remove a cell entirely, use Clear cell — it sets tile 0 and won't render.
      • Flipped tiles don't cost a tile slot. Unlike the Backgrounds page. Use Flip H freely for mirrored enemies.
      • Right-click a cell to stamp a tile. Highlight a tile in the tileset, then right-click any cell in the composite sprite to drop that tile in — fast way to fill a sprite without switching modes.
      • The tileset highlight is independent of the selected cell. Selecting a cell no longer changes the tileset highlight, so you can freely browse tiles without losing your place. Use left-click in the tileset to actually assign.
      • "Where is this tile used?" Right-click the tile in the shared tileset — the editor will jump to the first sprite that uses it. Handy when you're hunting for where a specific shape lives.
      • Dark colours hard to see? Use the BG: picker (🌑 / 🌗 / 🌕) to change the transparent-pixel background. Picking "Light" makes blacks and dark purples pop.
