[Reconciliation notes](#reconciliation-notes)), and the whole suite
re-run.

## Fixed bugs

Severity tags: 🔴 critical · 🟠 high · 🟡 medium · ⚪ low.

### 1. Vertical / 2×2 scrolling and palette corruption

These are the long-standing reported pupil bugs (bug-list items 9, 16,
29). Three independent root causes:

- 🟠 **`scroll_apply_ppu` used a 256-px vertical nametable boundary.**
  NES nametables are **240 px** tall. `if (cam_y & 0x100)` never fired
  for the reachable `cam_y == 240`, so the bottom screen (NT2) was never
  selected, and a Y-scroll of 240–255 (illegal) was written, rendering
  garbage. Now folds `cam_y` into a 0–239 offset plus a 240-px band whose
  low bit selects NT2.
  [`steps/Step_Playground/src/scroll.c`](../../steps/Step_Playground/src/scroll.c) — `scroll_apply_ppu`.

- 🔴 **Row streamer wrote into the attribute table.**
  `scroll_stream_prepare` mapped world rows to VRAM with mod-32 maths
  (`(row & 0x20)` / `(row & 0x1F) * 32`), but a nametable is 30 rows tall.
  Row 30 landed at `$23C0` (NT0's attribute table) and every streamed-in
  bottom row was shifted up by two — corrupting palettes as the camera
  scrolled down. Now maps rows 0–29 → NT0 (`$2000`) and 30–59 → NT2
  (`$2800`) to match `load_world_bg`.
  [`steps/Step_Playground/src/scroll.c`](../../steps/Step_Playground/src/scroll.c) — `scroll_stream_prepare`.

- 🟠 **World attribute table sized one screen-row too short.**
  `_world_nametable` sized the attribute array as `(rows+3)//4` rows. A
  30-tile screen is 7.5 attribute rows, so a 1×2 / 2×2 world's bottom
  screen read **past the end of `bg_world_attrs[]`** and from mis-aligned
  source rows → wrong palettes. Now emits a full **8-attr-row band per
  screen**, each derived from that screen's own tile rows, laid out at
  the stride the engine indexes with. **Verified byte-identical for 1×1
  and 2×1** (only the previously-broken 1×2 / 2×2 layouts change).
  [`tools/playground_server.py`](../../tools/playground_server.py) — `_world_nametable`.

> Within the 2×2 cap, `load_world_bg` pre-loads every visible nametable,
> so with the three fixes above the streamer only ever re-writes
> already-correct data into the correct nametable. **Resolved for every
> world the editor can currently produce.**

### 2. NES engine — `main.c` + `platformer.c` (kept byte-identical)

Every edit here was applied identically to both
[`steps/Step_Playground/src/main.c`](../../steps/Step_Playground/src/main.c)
and
[`tools/tile_editor_web/builder-templates/platformer.c`](../../tools/tile_editor_web/builder-templates/platformer.c)
so the byte-identical-ROM invariant holds.

- 🟠 **Player teleport on tall/scrolled screens.** The landing snap
  (`py = (unsigned char)((foot_row << 3) - …)`) and the ladder climb
  temporaries (`unsigned char new_top` / `new_foot`) truncated world-Y to
  8 bits, flinging the player to the top of the world once `py > 255`.
  Now computed in `pxcoord_t` (u16 under `SCROLL_BUILD`, u8 otherwise — so
  1×1 builds stay byte-identical). Player 2's landing snap fixed the same
  way.
- 🟠 **OAM shadow-buffer overflow.** The scene-sprite, animated-sprite and
  HUD-heart fill loops wrote with no upper bound, scribbling past the
  256-byte `oam_buf` into adjacent RAM when a scene exceeded the NES's 64
  hardware sprites. Each 4-byte write group is now guarded
  (`if (oam_idx <= 252)`). The bounded player loops are left unguarded so
  they stay byte-identical.
- 🟡 **Dialogue snapped the camera to (0,0).** `draw_text` /
  `clear_text_row` unconditionally reset `PPU_SCROLL = 0`, jerking the
  camera to the world origin on every dialogue frame in scrolling games.
  Now restores the camera via `scroll_apply_ppu()` under `SCROLL_BUILD`.
  *(Partial fix for bug item 28 — text still lands at fixed NT0 coords;
  making it track the camera is left for the dialogue rework.)*
- 🟡 **Multi-tile scene sprites fell through platform edges.** Scene-sprite
  gravity probed only the left foot column. Now probes both columns, like
  the player.
- 🟠 **Doors to backgrounds 4–9 loaded room 0.** `load_background_n` only
  had `case 0..3`; the schema/validator/emitter already supported up to 9
  rooms. Added the missing `case 4..9`, each gated by `#if BG_COUNT > N`
  (byte-identical-safe).

### 3. Browser editor — pages

- 🔴 **Code page "Play in NES" was completely broken.** `onRom` re-encoded
  the decoded ROM bytes to a base64 *string* and handed that to jsnes,
  which threw "Not a valid NES ROM." every time. Now passes a proper
  binary string and removes the dead helpers.
  [`tools/tile_editor_web/code.html`](../../tools/tile_editor_web/code.html).
- 🟠 **Code page always bounced pupils with custom C to the Builder.** The
  redirect IIFE read a storage key that the v2 migration deletes. Now
  resolves the active project through the v2 catalog (with a legacy

- 🟠 **Doors led to the wrong screen after delete/duplicate.** Deleting or

  helper called from both handlers.
  [`tools/tile_editor_web/index.html`](../../tools/tile_editor_web/index.html).

- 🟡 **Undo mid drag-move re-stamped lifted pixels.** `afterStateReplaced`

  [`sprites.html`](../../tools/tile_editor_web/sprites.html).
- 🟡 **Audio page told pupils to drag `.s` files but had no drop handler**
  (dropping navigated away and lost work). Added a drop handler routing
  song/sfx files to the existing upload paths.
  [`tools/tile_editor_web/audio.html`](../../tools/tile_editor_web/audio.html).
- ⚪ **Animation created with invalid empty role/style tags** desynced the
  tag dropdowns; now uses valid defaults (`player`/`custom`).
  [`sprites.html`](../../tools/tile_editor_web/sprites.html).
- ⚪ **Accessibility prefs silently failed to persist** on the gallery,
  audio and builder pages (IIFE-scoped `Storage` hid it from `a11y.js`).

  [`gallery.html`](../../tools/tile_editor_web/gallery.html),

## Remaining work

Nothing here blocks current pupils — these are either future architectural
work or low-value/high-risk clean-ups deliberately left alone.

### Deferred — scroll engine beyond the 2×2 cap (plan §T3.1 / §T3.2)

The scroll **streamer** (mid-game column/row fetch, attribute streaming,
and per-background tile swap after a door) is still incomplete for worlds
**larger than 2×2**. The editor cannot currently create such worlds, and
within the 2×2 cap `load_world_bg` pre-loads everything, so this is latent.
Three confirmed defects sit here:

1. **Row streaming uses the wrong source columns in a 2×2 world.** A
   32-tile burst from columns 0–31 can't fill a 64-tile-wide visible row,
   and it isn't shifted into NT1/NT3 by `cam_x`. *(Within the 2×2 cap this
   is masked by the up-front pre-load; it only bites once worlds exceed
   two screens.)*
2. **Post-door rooms stream stale tiles.** `scroll_stream` always reads the
   originally-selected background's `bg_world_tiles[]`. Needs per-bg world
   arrays + a `scroll_set_active_bg` swap, and a re-prime of `prev_cam`.
3. **No attribute streaming for scrolled-in columns/rows.** Only needed
   once worlds exceed 2×2 (attributes for ≤2×2 are loaded up front).

These should land together as the §T3.1 / §T3.2 rewrite, ideally with a
pathological-2×2 PPU regression test. Detailed symptom checklists are in
[`../feedback/recently-observed-bugs.md`](../feedback/recently-observed-bugs.md)
(items 9 / 29).

### Deferred — `screen-shake-on-landing` snippet (cosmetic)

The snippet writes `PPU_SCROLL` during active render, where the engine
immediately overwrites it, so no shake is ever visible. A correct fix needs
a shared shake-offset global that the engine adds to its post-vblank scroll
write (in `lessons/03-magic-button.c`, the playground non-scroll path, and
`scroll_apply_ppu`). Deferred to avoid threading engine state through the
byte-identical baseline for a cosmetic effect.
[`snippets/screen-shake-on-landing.c`](../../snippets/screen-shake-on-landing.c).

### Deferred — `nes.cfg` CHR-bank / iNES-header mismatch (conformance)

[`steps/Step_Playground/cfg/nes.cfg`](../../steps/Step_Playground/cfg/nes.cfg)
declares a 16 KB CHR region while the iNES header says 8 KB, and 8 KB of it
is the unused `NESfont`. The ROM loads correctly in every emulator and
flashcart in use; a clean fix means dropping `NESfont` (touches the cc65
crt0 and the audio crt0) or post-patching the header byte. The link/crt0
risk outweighs the benefit, so deferred.

### Dialogue follow-up (bug item 28)

The camera-snap half is fixed. Making dialogue text *appear in the correct
on-screen position while the world is scrolled* (write into the visible
nametable instead of fixed NT0 coordinates, or a sprite-0/scanline split
for a fixed HUD line) is a larger change left for the dialogue rework.

## Reconciliation notes

A few findings were interdependent and were resolved as a set rather than
literally as written:

- **Server attr layout vs. engine attr index** — fixed server-side only
  (see "Not applied" above).
- **Ladder down-probe** — one reviewer wanted the temporary widened to
  `pxcoord_t`, another wanted it replaced with a direct row index. Chose
  the widen-the-type form because it preserves byte-identical 1×1 output
  while still fixing the truncation.
- **`main.c` ↔ `platformer.c`** — every engine fix was applied to both
  files in lockstep; the dialogue edit used targeted (not global) replaces
  so it would not also rewrite `load_background_n`'s scroll reset, which is
  a separate (door) concern.
